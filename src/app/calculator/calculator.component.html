Technical design specification

It would be a traditional calculator, but it should have two special details: undo button and realization of a command pattern inside.

Usually Command pattern means encapsulating a request as an object, thereby letting you parameterize clients with different requests, queue or log requests, and support reverse operations.

So, there is a calculator with 4 basic operations: add, subtract, multiply and divide. Each operation is encapsulated by a Command object. The calculator maintains a stack of commands. Each new command is executed and pushed onto the stack. When reverse operation request arrives, it simply pops the last command from the stack.

Realization

There is a Command pattern realization.
It encapsulates the calling of a method as an object.
There are four function for simple math operations:
add, subtraction, multiplication, division.
There is a class Command which receives execute, undo and value and writes it in to the " this".
And there are some wrapper-classes which return our commands and actually they are used as a Factory.
And there is a Calculator class.
It stores array of commands in this._commands and stores current value of calculation in this._current.
And it has execute method which writes command result in current.
