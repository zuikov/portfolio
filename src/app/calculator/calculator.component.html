Technical design specification

It would be a traditional calculator, but it should have two special details: undo button and realisation of a command pattern inside.

Usuallly Command pattern means encapsulating a request as an object, thereby letting you parameterize clients with different requests, queue or log requests, and support undoable operations.

So, there is a calculator with 4 basic operations: add, subtract, multiply and divide. Each operation is encapsulated by a Command object. The calculator maintains a stack of commands. Each new command is executed and pushed onto the stack. When an undo request arrives, it simply pops the last command from the stack.

Realisation

Pattern Command encapsulate the calling of a method as an object Есть четыре функции, которые описывают простейшие математические действия: add, substraction, multiplication, division. Есть класс Command, который принимает execute, undu and value and writes it in to the "this". Есть классы-обертки, которые возвращают нашу команду и по сути являются фабрикой. Есть класс калькулятор который хранит массив комманд в this._commands и текущее значение вычислений в this._current У него есть метод execute, который записывает в current результат выполнения команды.
